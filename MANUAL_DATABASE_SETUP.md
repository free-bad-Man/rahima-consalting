# Ручная настройка базы данных (без Vercel CLI)

## Шаг 1: Получите переменные из Vercel Dashboard

1. Зайдите на https://vercel.com/dashboard
2. Выберите ваш проект **rahima-consalting**
3. Перейдите в **Settings** → **Environment Variables**
4. Найдите переменную **`POSTGRES_URL`** или **`DATABASE_URL`**
5. Скопируйте её значение (нажмите на значок глаза, чтобы показать значение)

## Шаг 2: Обновите .env.local

Откройте файл `D:\work\.env.local` и замените/добавьте:

```env
DATABASE_URL="<скопированное значение POSTGRES_URL из Vercel>"
```

**Важно:** Используйте значение `POSTGRES_URL` (без pooling), а не `PRISMA_DATABASE_URL` (с pooling) для миграций.

## Шаг 3: Примените миграции

```powershell
npx prisma migrate dev --name init
```

Или если миграции не нужны, просто примените схему:

```powershell
npx prisma db push
```

## Шаг 4: Проверьте

После применения схемы попробуйте авторизоваться через Google на вашем сайте на Vercel - должно работать! ✅

---

## Альтернатива: Использовать переменные напрямую из Vercel

Если на Vercel уже настроены правильные переменные окружения, можно просто задеплоить проект - миграции применятся автоматически при сборке (если добавить в package.json).

Но лучше применить миграции локально один раз, чтобы создать таблицы.
