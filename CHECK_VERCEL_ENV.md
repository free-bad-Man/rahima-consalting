# Проверка переменных окружения на Vercel

## Проблема
База данных `db.prisma.io` недоступна. Это может быть потому что:
1. База данных еще активируется
2. Нужно использовать переменные из Vercel напрямую

## Решение

### Вариант 1: Использовать переменные из Vercel напрямую

1. **Зайдите в Vercel Dashboard**:
   - Ваш проект → **Settings** → **Environment Variables**

2. **Проверьте, какие переменные там есть**:
   - `DATABASE_URL`
   - `POSTGRES_URL`
   - `PRISMA_DATABASE_URL`

3. **Скопируйте значение `POSTGRES_URL`** (или `DATABASE_URL`, если `POSTGRES_URL` нет)

4. **Обновите `.env.local`**:
   ```env
   DATABASE_URL="<скопированное значение из Vercel>"
   ```

### Вариант 2: Подождать активации базы данных

Если база создана недавно (36 минут назад), возможно она еще настраивается:
- Подождите 10-15 минут
- Попробуйте снова

### Вариант 3: Проверить статус базы в Prisma Data Platform

1. Зайдите в Prisma Data Platform (как на скриншоте)
2. Проверьте статус базы данных
3. Убедитесь, что она "Available" (зеленая точка)

### Вариант 4: Использовать Prisma Accelerate (для работы приложения)

Для работы приложения можно использовать Prisma Accelerate:
- Строка: `prisma+postgres://accelerate.prisma-data.net/?api_key=...`
- Но для миграций (`prisma db push`) нужна обычная PostgreSQL строка

Попробуйте:

2. Проверить переменные окружения на Vercel
3. Использовать значение `POSTGRES_URL` или `DATABASE_URL` из Vercel
